# 预处理规则

## 目录原则

原始卷宗不复制进工作区。用户只提供原始 PDF 文件夹路径，脚本只记录原始路径、文件名、大小、页数、哈希或修改时间。工作区只保存派生文件。

推荐结构：

```text
cases/[案件名]/
├── 00_manifest/
├── 01_working/
├── 02_pages/
├── 03_ocr_json/
├── 04_ocr_text/
├── 05_index/
└── 06_outputs/
```

## 处理顺序

1. 清点 PDF：文件名、大小、页数、是否加密、是否可打开、是否已有文本层。
2. 密码处理：有打开密码时由用户提供密码。不得破解密码。只对工作副本解密。
3. 水印处理：仅当水印明显遮挡正文或影响 OCR 时处理。去水印副本必须标注“仅供 OCR/检索，不作为原件”。
4. 渲染页面：按需把 PDF 页渲染为图片。页图是可再生缓存，可在 OCR 后清理。
5. OCR：优先调用本机 Umi-OCR API，默认 `http://127.0.0.1:1224/api/ocr`。禁止默认调用云端 OCR 或在线 AI 服务。
6. 落地：每页保存 OCR JSON、纯文本、页级索引和处理状态。

## OCR 限制

OCR 结果只用于检索和初筛。下列事项必须回看原始页面：

- 签名、捺印、印章、手写内容；
- 卷宗页码、页码跳号、缺页；
- 讯问起止时间、地点、讯问人、记录人、同步录音录像说明；
- 认罪认罚具结书和值班律师签名；
- 扣押、提取、封存、送检、鉴定、移送链条；
- 表格、清单、截图、聊天记录、银行流水等版面敏感材料。

## 大体积卷宗策略

- 先清点，后 OCR；先索引，后分析。
- 支持断点续跑：已有 JSON/TXT 不重复 OCR，失败页单独记录。
- 不把全卷 OCR 文本一次性读入模型上下文；通过 SQLite、CSV 和关键词分批检索。
- 15G 级卷宗优先按卷名、文件夹、材料类型或页码范围分批处理。
